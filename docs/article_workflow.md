# Жизненный цикл статьи и управление доступом

Этот документ описывает полный жизненный цикл статьи в системе, а также права доступа для каждой роли пользователя.

## 👥 Роли пользователей

1.  **Анонимный пользователь (Гость)**
    -   Не может создавать, изменять или удалять статьи.
    -   Может просматривать только статьи со статусом `published`.

2.  **Зарегистрированный пользователь (`reader`, `author`, `editor`)**
    -   **Создание**: Может создавать статьи. Статье автоматически присваивается статус `draft`.
    -   **Обновление**: Может обновлять **только свои** статьи.
    -   **Архивация**: Может архивировать **только свои** статьи (статус `archived`). Это обратимое действие.
    -   **Смена статуса**:
        -   Может отправить свою статью на проверку (статус `draft` -> `review`).
        -   Может вернуть статью с проверки обратно в черновики (статус `review` -> `draft`).

3.  **Администратор (`admin`)**
    -   **Создание и обновление**: Может создавать и обновлять **только свои** статьи, как и обычный автор.
    -   **Смена статуса**: Может изменять статус **любой** статьи на любой другой (например, опубликовать статью со статусом `review`, вернуть статью из архива и т.д.).
    -   **Полное удаление**: Обладает правом безвозвратно удалять любую статью из базы данных (будет реализовано отдельной мутацией).

## ♻️ Жизненный цикл статусов статьи

Статья может находиться в одном из следующих состояний:

-   `draft`: Черновик. Виден только автору и администраторам.
-   `review`: На проверке. Виден автору и администраторам.
-   `published`: Опубликовано. Виден всем.
-   `archived`: В архиве. Виден только автору и администраторам.

### Схема переходов статусов:

-   **Пользователь (Автор):**
    -   `CREATE` -> `draft`
    -   `draft` -> `review` (через мутацию `requestReview`)
    -   `review` -> `draft` (через мутацию `revertToDraft`)
    -   `draft` | `review` | `published` -> `archived` (через мутацию `archiveArticle`)

-   **Администратор:**
    -   Может переключить статью в **любой** статус из **любого** другого (через мутацию `setArticleStatus`). Например:
        -   `review` -> `published` (публикация статьи)
        -   `archived` -> `published` (восстановление из архива)
        -   `published` -> `draft` (отзыв публикации для доработки)

##  Mutations API

### Для всех зарегистрированных пользователей:

-   `createArticle(input)`: Создает статью со статусом `draft`.
-   `updateArticle(id, input)`: Обновляет свою статью.
-   `archiveArticle(id)`: Архивирует свою статью.
-   `requestReview(id)`: Отправляет свою статью на проверку.
-   `revertToDraft(id)`: Возвращает свою статью с проверки в черновики.

### Только для администраторов:

-   `setArticleStatus(id, status)`: Устанавливает любой статус для любой статьи.
-   `permanentlyDeleteArticle(id)`: (Будет добавлено позже) Безвозвратно удаляет статью.
