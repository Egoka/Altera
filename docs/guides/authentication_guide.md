# Руководство по тестированию авторизации

Этот документ описывает, как протестировать полный цикл Magic Link авторизации с помощью GraphQL-клиента (например, Insomnia, Postman или GraphiQL).

## Шаг 1: Запрос Magic Link

Первый шаг — инициировать вход для пользователя с определенным email. Если пользователя с таким email не существует, он будет создан автоматически.

Отправьте следующую GraphQL-мутацию:

```graphql
mutation RequestMagicLink {
  requestMagicLink(email: "test@example.com")
}
```

## Шаг 2: Получение токена из консоли

После выполнения мутации, откройте терминал, в котором запущен сервер. Вы увидите сгенерированную "магическую ссылку":

```
✨ Magic Link for test@example.com: http://localhost:3000/auth/verify?token=a1b2c3d4...
```

Скопируйте **значение токена** из параметра `token` в этой ссылке. Это ваш одноразовый токен для входа.

## Шаг 3: Обмен токена на JWT

Теперь используйте полученный токен, чтобы получить долгоживущие `accessToken` и `refreshToken`. Эти JWT будут использоваться для аутентификации последующих запросов.

Отправьте следующую мутацию, подставив ваш токен:

```graphql
mutation VerifyMagicLink {
  verifyMagicLink(token: "ВАШ_СКОПИРОВАННЫЙ_ТОКЕН") {
    accessToken
    refreshToken
    user {
      id
      name
      email
    }
  }
}
```

В ответе вы получите пару токенов и информацию о пользователе. Сохраните `accessToken` для следующего шага.

## Шаг 4: Выполнение аутентифицированного запроса

Теперь вы можете делать запросы к защищенным эндпоинтам. Для этого необходимо передавать `accessToken` в заголовке `Authorization`.

1.  **Настройте заголовок:**
    *   **Ключ:** `Authorization`
    *   **Значение:** `Bearer ВАШ_ACCESS_TOKEN` (обратите внимание на `Bearer ` с пробелом).

2.  **Отправьте запрос** к защищенному эндпоинту, например, для получения данных своего профиля:

```graphql
query GetMyProfile {
  me {
    id
    email
    role
  }
}
```

### Ожидаемые результаты

*   **Успех:** Если токен валиден, вы получите данные пользователя (`id`, `email`, `role`).
*   **Ошибка:** Если заголовок `Authorization` отсутствует или токен недействителен, сервер вернет ошибку `Authentication required.`.
