# Документация проекта Altera

Эта папка содержит всю документацию проекта, организованную по категориям для удобства навигации.

## 📁 Структура документации

### 📂 guides/
Руководства и стандарты разработки:
- `authentication_guide.md` - руководство по тестированию авторизации
- `admin_query_standards.md` - стандарты для админ-запросов
- `article_workflow.md` - рабочий процесс статей

### 📂 reports/
Отчеты о рефакторинге и исправлениях:
- `status_filter_refactoring_report.md` - отчет о рефакторинге фильтров статуса
- `duplicate_query_fixes_report.md` - отчет об исправлении дублирующихся запросов
- `search_parameter_refactoring_report.md` - отчет о рефакторинге параметров поиска
- `graphql_schema_fixes_report.md` - отчет об исправлениях GraphQL схемы
- `admin_panel_corrections_report.md` - отчет об исправлениях админ-панели
- `admin_panel_implementation_report.md` - отчет о реализации админ-панели

### 📂 architecture/
Архитектурная документация:
- `database_architecture.md` - архитектура базы данных

### 📂 status/
Статус реализации и структура:
- `api_implementation_status.md` - статус реализации GraphQL API
- `graphql_queries_structure.md` - структура GraphQL запросов

### 📂 queries/
Тестовые GraphQL запросы:
- `admin/` - запросы для админ-панели
- `author/` - запросы для авторов
- `common/` - общие запросы

## 🎯 Назначение документации

- **Guides** - для разработчиков, которые хотят понять как работать с проектом
- **Reports** - для отслеживания изменений и рефакторинга
- **Architecture** - для понимания структуры системы
- **Status** - для мониторинга прогресса разработки
- **Queries** - для тестирования и документирования API

## 📝 Рекомендации по использованию

1. Начните с `guides/` для понимания стандартов
2. Изучите `architecture/` для понимания структуры
3. Проверьте `status/` для актуального состояния
4. Используйте `queries/` для тестирования API
5. Читайте `reports/` для понимания истории изменений
